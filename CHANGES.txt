RCS QA 系统 — 本次会话改动总结
====================================

【项目概述】
RCS QA — 风控告警逻辑质检系统。
Vue 3 + Element Plus 前端，Node.js + Express + MongoDB 后端，Python rc_sync_service.py 自动同步。


===========================================================
一、新增功能
===========================================================

1. 存提差环比（typeId 9）告警类型
   - backend/models/index.js：TestList.records 新增 5 个字段
     （depositAmount、withdrawalAmount、lastNetflowAmount、lastDepositAmount、lastWithdrawalAmount）
     default: null（null=接口未返回，0=值为零）
   - backend/routes/syncCache.js：TYPE_ID_MAP 加 netflow-additional-present-day: 9
   - frontend/src/logic/alertTypes.js：加 typeId 9 映射、PAGE_TITLES、val1 标签
   - frontend/src/logic/netflowCompLogic.js：新建，告警结果 = (Last存提差 − 当前存提差) ≥ Y → TRUE
   - frontend/src/views/TestView.vue：扩展支持 typeId 9，新增 5 个数值列 + 下降金额列 + 告警结果逻辑
   - frontend/src/views/CaptureView.vue：加 typeId 9 字段映射配置（接口地址、9 个字段）
   - frontend/src/views/ConfigView.vue：加 typeId 9（检查间隔 X + 下降阈值 Y 两个参数）
   - frontend/src/router.js：/test/9 自动走 TestView，移除独立 /test/netflow-comp
   - frontend/src/App.vue：侧边栏加「存提差环比」→ /test/9
   - NetflowCompView.vue：保留文件但已不被路由（可删除）

2. 数据导出工具新增选项（frontend/src/views/QAConfigView.vue）
   - 接口：新增 allGameProfitAlerts（游戏盈利 CG）
   - 告警类型：新增 netflow-additional-historical（存提差同比）
               netflow-additional-present-day（存提出环比）
               game-profit（游戏盈利 CG）
   - 接口切换时告警类型自动联动过滤（只显示该接口下的有效类型）


===========================================================
二、重要修复
===========================================================

3. 同步数据抓取逻辑重写（backend/routes/syncCache.js formatSyncRecord）
   - 所有字段路径硬编码，不再依赖 captureConfig 运行时缓存
   - typeId 9 额外字段（depositAmount 等）从 RC 顶层字段直接读取
   - typeId 1-4：val2 = upperBoundary / amountFactor（计算平均值）
   - typeId 5-7：val2 = upperBoundary
   - devResult：evalDevResult(item.alertNumber)，有告警号即 TRUE
   - 未抓到的字段返回 null（前端显示 ⚠ 未抓到），与值为 0 区分
   - /sync-cache/:typeId 改为 async，首次调用自动 refreshCaptureConfigCache()

4. rc_sync_service.py — 清除日期过滤器（新增 clear_date_filters 函数）
   - 用 JavaScript TreeWalker 扫描页面文本节点，找含日期格式的元素
   - 精确定位右侧 X 按钮（水平距离 0~80px，垂直对齐，尺寸 ≤44px）
   - 发送 mousedown + click + mouseup 兼容 React 合成事件
   - 初始启动和每次同步触发前都调用，确保 Search 返回全量数据
   - 顺序保证：三个页面全部「清除 → Search」完成后，poll_trigger_loop 才启动

5. rc_sync_service.py — 凭证多环境支持
   - 测试站 qamanual（platform88.me），正式站 jordan（platform10.me）
   - 根据 --url 参数自动选择对应账号和 OTP secret
   - .env 加 RC_PROD_USERNAME / RC_PROD_PASSWORD / RC_PROD_OTP_SECRET

6. rc_sync_service.py — 旧数据过滤
   - 新增 filter_recent(records, days=14)
   - 推送前过滤 14 天以外的旧数据，防止正式站日期为空时污染缓存

7. rc_sync_service.py — response handler 注册时机
   - 改为导航完成后再注册，避免捕获初始加载的旧数据
   - 正式站默认无日期过滤会返回大量历史数据，此改动防止污染

四、数据库优化
===========================================================

12. backend/models/index.js 全面重构

    - depositAmount 等字段 default: undefined → default: null

13. backend/server.js — 启动迁移（runStartupMigrations）
    - 自动 drop gameprofitconfigs 孤儿集合
    - captureconfigs.endpoint 空字符串统一改为 null
    - testlists 的 typeId 9 扩展字段补 null

14. backend/routes/testLists.js / gameProfitLists.js
    - 保存前检查 records 数量，超过 5000 条返回 400 错误

15. backend/routes/captureConfig.js
    - endpoint 空字符串转 null，fields 过滤空路径条目

16. backend/reset_db.js 重写

      node backend/reset_db.js          # 全清（含配置），有确认提示
      node backend/reset_db.js --data   # 只清测试数据，保留配置
      node backend/reset_db.js --force  # 跳过确认直接清
    - 显示清理前条数、结果，自动 drop 孤儿集合

===========================================================
关键架构说明
===========================================================

决策                              原因
--------------------------------  ------------------------------------------
存提差环比复用 TestView            降低维护成本，共享同步/批量/统计基础设施
formatSyncRecord 硬编码字段路径    captureConfig 缓存有加载时机问题，hardcode 更稳定
captureConfig 只作 UI 文档用途     数据抓取靠代码，配置页告诉用户「字段来自哪里」
null vs 0 区分                    帮助用户排查字段路径配置错误
rc_sync_service 清除日期全量抓取   正式站页面默认无日期，不清除会漏掉历史告警数据
handler 注册在导航后               避免捕获初始加载的旧数据污染同步缓存


===========================================================
当前已知待完善
===========================================================

- 告警配置中「开发中」的字段（持续时间值、金额、配置生效时间）是占位，正常显示即可，不需处理
- captureConfig 在 UI 里配置的路径，如果路径错误前端会显示「⚠ 未抓到」，但没有路径格式验证
- synccaches 集合的 updatedAt 是 String 类型，MongoDB TTL 索引需要 Date 类型才能自动过期，
  目前只加了普通索引，自动清理需要手动运行 reset_db.js --data
