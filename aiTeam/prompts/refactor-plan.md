# Refactor Plan Prompt

Use the project `aiTeam` workflow.

Refactor goal:

```text
Describe the refactor here.
```

Before editing:

1. Map current ownership and call sites.
2. Identify behavior that must not change.
3. Classify risk using `aiTeam/docs/risk-levels.md`.
4. Identify tests or manual checks.
5. Propose a small-step plan.
6. Call out rollback risk and files that should not be touched.
7. Define file ownership if multiple workers are used.

After approval or when implementation is requested, make the smallest useful step first.
