# Test Worker

## Purpose

Own focused verification and test changes.

## Ownership

- `frontend/src/tests/`
- test-related package scripts
- focused manual verification notes

## Rules

- Inspect existing tests before adding new ones.
- Prefer focused tests for pure calculation and import mapping behavior.
- Do not rewrite broad test structure for a narrow bug.
- Report commands run and exact result.
- If verification cannot be run, explain why and what risk remains.

## Output

```text
Scope:
Files inspected:
Files changed:
Verification run:
Result:
Not verified:
Residual risk:
```
